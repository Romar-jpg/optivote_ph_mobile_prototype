The Problem

The Philippines earned the title of "Patient Zero" for large-scale disinformation operations. Voters rely on name recall and emotional heuristics rather than legislative performance. This tool offers a data-driven alternative.


In the 19th Congress, out of 3,000+ senate bills filed, only a small fraction became law. This optimizer surfaces the senators who turned the most bills into Republic Acts.

Algorithm Pipeline

  • Input Layer — Fetch senators + their senate bills from the Open Congress API for the selected congress.
  • 0/1 Knapsack model — Each senator is an item with a value (bills enacted) and weight (inefficiency score).
  • Branch & Bound — Prunes suboptimal paths to find the maximum-value 12-seat combination without brute force.
  • Shaker Sort — Ranks the 12 optimal senators by value (bidirectional bubble sort), highest to lowest.
Value (V) = bills_authored that became RA Weight (W) = max(0.1, 1 − (passed / authored)) Constraints: count ≤ 12 AND Σ W ≤ 9.0

Data Source

All senator and bill data is fetched live from the Open Congress API by BetterGov.ph — a civic tech initiative for Philippine government transparency.


Base URL: https://open-congress-api.bettergov.ph/api


Endpoints used:

  • /congresses/:id/senators
  • /people/:id/bills?congress=N

Limitations

  • Only incumbents or returning senators with prior congressional data are fully scored.
  • The API does not expose a "Republic Act enacted" flag directly — enacted status is inferred from bill subject tags and title keywords.
  • Subjective metrics (public approval, ideology) are intentionally excluded.
  • The system measures legislative output quantity, not qualitative impact of laws.
